java-垃圾回收-面试题

2019-04-02

finalize()方法的作用是否与c++的析构函数相同？

  1. 不同。析构函数调用确定，finalize()不确定
  2. 方法随时可能终止

~

会将未被引用的对象放在F-Queue队列

可以用来给予对象最后的复活机会

强 软 弱 虚引用？

强引用

特点:

1. 最普遍的引用方式 Object obj = new Object();
2. 对象设置为null来使其被回收
3. 就算抛出OutOfMemoryError也不会回收强引用对象

软引用

特点：

  1. 指对象处于有用但没必要的状态
  2. 内存空间不足时会被回收

可以用来做高速缓存

弱引用

特点：

  1. 非必须的对象
  2. GC时被回收

被回收的概率不大，因为GC优先级很低

适合用于偶尔被使用且不影响垃圾收集的对象

虚引用

特点：

  1. 不会决定对象的生命期
  2. 随时可能被回收
  3. 必须使用引用队列

可以用于跟踪对象被GC回收的活动。哨兵作用

引用队列(ReferenceQueue)

特点：

  1. 无实际存储结果，存储逻辑依赖于内部节点之间联系(类似于链表)

可以存储关联的 软 弱 虚 引用

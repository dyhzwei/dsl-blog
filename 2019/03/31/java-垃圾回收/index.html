java堆

判断垃圾的标准

  • 引用计数法
  • 可达性分析
引用计数法

  • 对象的引用计数器：被引用->+1 ; 完成引用->-1 (方法运行结束后，局部变量引用会自动释放)

  • 为0就当做垃圾

  • 优点：效率高，很少打断程序执行

  • 缺点：若循环引用，则无法回收，内存泄漏。看下例。

可达性分析

  • 从GCroot 是否有一条路径能到达对象

  • GCroot对象：

    • 虚拟机栈中的引用对象
    • 方法区中的常量中的引用对象
    • 方法区中的类静态变量中的引用对象
    • 本地方法栈中的JNI（Native方法）的引用对象
    • 活跃线程的引用对象

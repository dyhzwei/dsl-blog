java-垃圾回收

2019-03-31

gc主要目标区域

java堆

判断垃圾的标准

  • 引用计数法
  • 可达性分析

引用计数法

思路：对象的引用计数器：被引用->+1 ; 完成引用->-1 (方法运行结束后，局部变量引用会自动释放 为0就当做垃圾

优点：效率高，很少打断程序执行

缺点：若循环引用，则无法回收，内存泄漏。看下例。

可达性分析

思路： 从GCroot 是否有一条路径能到达对象

  • GCroot对象：
    1. 虚拟机栈中的引用对象
    2. 方法区中的常量中的引用对象
    3. 方法区中的类静态变量中的引用对象
    4. 本地方法栈中的JNI（Native方法）的引用对象
    5. 活跃线程的引用对象

垃圾回收算法

标记清除（Mark and Sweep）

缺点：会产生碎片

复制（Copying）

思路：分为两个区域，一个用于分配，一个用于接纳复制的存活对象。随后角色调换
特点：

  1. 不会产生碎片
  2. 顺序分配内存，简短高效
  3. 内存使用率低
  4. 相性：对象存活率低~年轻代

标记整理（ Compacting）

思路： 存活对象向前紧凑，按内存地址排序，末端地址后的全部回收
特点：

  1. 无内存碎片
  2. 相性：存活率高~老年代

分代回收（Generational Collection）

思路：混合型算法，寿命不同采用不同策略
MinorGC + MajorGC = FullGC
年轻->MinorGC ; 老年->MajorGC+MinorGC

发展：
JDK1.6、1.7 年轻，老年，永久
JDK1.8 及以上 年轻，老年

年轻代MinorGC

复制法

  1. 首次Eden复制到from；后续是Eden+from 复制到 to 区

  2. 回收，存活对象年龄+1 若超过15则进入老年代

  3. from 和 to对调

总结·混入老年代方法

  1. 存活次数（年龄）超过阈值
  2. 复制时Survivor区放不下
  3. 新生成的大对象size超阈值

调优参数
Eden和Survivor的比值，默认8:1=> -XX:SurvivorRation:
老年代年轻代大小比例 => -XX:NewRatio:
设置升级年龄阈值=> -XX:MaxTenuringThreshold:
大对象size阈值=> -XX:PretenuerSizeThreshold:

老年代MajorGC

策略：标记-清理 标记-清理

一般进行老年代GC的时候都是因为先前的年轻代GC，整体成为FullGC

FullGC

触发条件：

  1. 老年代空间不足
  2. JDK1.6 1.7 永久代空间不足~所以1.8减少了GC频率
  3. 年轻代晋升的平均size大于剩余空间 （根据平均size估计，新分配可能不够）
  4. 程序员调用 System.gc()
  5. 用RMI 进行RPC或管理的JDK应用，每1h执行一次FullGC

~CMS 回收器

Stop-the-World

概念：JVM为了执行GC而停止执行应用程序

特点：任何一种GC都会发生

优化主要就是指减少stw的时间

SafePoint

概念：对象引用关系不发生改变的点
出现：

  1. 方法调用
  2. 循环跳转
  3. 异常跳转

吞吐量

概念：运行用户代码时间/(运行用户代码时间+GC时间)

抉择

缩短用户停顿时间 or 增大吞吐量

Serial收集器

年轻代收集器

策略：复制

特点：

  1. 单线程收集，收集时，必须暂停所有工作线程
  2. 简单高效

~

JVM client模式下的默认年轻代收集器

-XX:UseSerialGC

ParNew收集器

年轻代收集器

策略：复制

特点：

  1. 多线程收集
  2. 其它行为和Serial收集器一样

~

多核才有利

默认并发数=核数

-XX:UseParNewGC

Parallel Scavenge收集器

年轻代收集器

策略：复制

特点：

  1. 高吞吐量
  2. 多线程

~

JVM Server模式下默认的年轻代收集器

默认并发数=核数

-XX:UseParallerGC
-XX:UseAdaptiveSizepolicy 调优任务交给jvm去完成

Serial Old收集器

老年代收集器

策略：标记整理

特点：

  1. 单线程收集，收集时，必须暂停所有工作线程
  2. 简单高效

~

JVM client模式下的默认年轻代收集器

-XX:UseSerialOldGC

Parallel Old收集器

老年代收集器

策略：标记整理

特点：

  1. 多线程
  2. 吞吐量优先

~

-XX:UseParallelOldGC

CMS收集器

老年代收集器

策略：标记清除

步骤：

  1. 初始标记， stw 找到所有和GCroot直接关联的对象
  2. 并发标记，并发执行追踪step1所有对象的关联对象
  3. 并发预清理，并发寻找晋升对象
  4. 重新标记，暂停虚拟机，扫描CMS堆剩余对象
  5. 并发清理
  6. 并发重置

step1 和 step4 会暂停

-xx：UseConcMarkSweepGC

G1收集器

年轻代 老年代收集器

策略：复制 标记整理

特点：

  1. 并行和并发
  2. 分代收集
  3. 无内存碎片
  4. 可预测的停顿
  5. 将整个java堆内存分成多个大小相等的Region进行管理
  6. 年轻代和老年代不再物理隔离

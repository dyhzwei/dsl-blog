java-多线程

2019-04-02

进程和线程

  1. 进程是资源分配的最小单位，线程是cpu调度的最小单位

  2. 进程是一个独立应用，拥有独立的地址空间

  3. 进程切换开销大

当一个线程执行出错后，所属的进程就死了

Java的进程线程

对操作系统的功能进行了封装

特点:

  1. 一个程序会产生一个进程，一个程序至少包含一个线程
  2. 一个进程对应一个jvm实例，多个线程共享jvm的堆
  3. 程序自动创建主线程
  4. jvm中还有其它线程如GC等

线程状态

可以查看Thread的源码

  • new ：创建后尚未启动
  • Runable：运行，包含Running和Ready
  • Waiting：无限等待，不会被分配CPU需要显示的被唤醒
  • Timed Waiting：限期等待，一定时间后系统自动唤醒
  • Blocked：等待获取排它锁
  • Terminated：结束执行

进入Waiting状态的途径

  1. wait()
  2. join()
  3. LockSupport.park()

进入Timed Waiting的途径

  1. sleep()
  2. wait(timeout)
  3. join(timeout)
  4. LockSupport.parkNanos()
  5. LockSupport.parkUntil()

notify和notifyAll的区别

锁池和等待池和对象锁一样，都是对象的里一个区域。

锁池(EntryList):当线程A占有了obj的锁，其它线程调用obj的sychronize方法和块时会被阻塞，进入obj的锁池等待。

等待池(WaitSet):当线程A调用obj.wait(),A就进入到了obj的等待池。

锁池相当于排队等着锁

等待池则是咸鱼区域，我调用obj.wait()了，我自闭了。你们自己去抢锁罢！

notify notifyAll
随机选取等待池中的一个对象进入锁池 所有在等待池中的线程都进入锁池
  • notify

说明：子线程1和子线程2 都调用myFlag.wait() 只有子线程3调用myFlag().notify()

因为notify()只会唤醒一个，因此结果是1和2中随机一个执行完毕，另一个无限等待
